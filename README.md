### 1. 프로젝트 소개
#### 1.1. 개발배경 및 필요성
최근 대형 IT·통신사를 비롯해 금융사, 쇼핑몰 등에서 발생한 연이은 대규모 개인정보 유출 사고는 단순히 정보 노출 문제를 넘어 스미싱, 피싱, 택배 사칭, 보이스피싱 등 심각한 2차 피해를 낳는 사회적 문제로 대두되었다. 이러한 상황에서 사용자가 유출 사고 이후 가장 먼저 필요로 하는 것은 단순한 유출 사실 확인을 넘어 자신의 상황에 맞는 구체적인 대응 시나리오이다. 일반 사용자는 기업으로부터 유출 안내문을 받더라도 법적 표현이나 기술적 용어가 포함된 복잡한 내용 탓에 정확히 어떤 정보가 유출되었고 이것이 어떤 피해로 이어지는지 판단하기 어렵다. 또한 기존의 개인정보 포털 등은 유출 여부 조회나 일반적인 보안 수칙 안내에만 집중되어 있어, 사용자가 받은 유출 통지문을 직접 해석하고 개인별 대응 순서를 정리해 주는 실질적인 가이드는 부족한 실정이다. 

#### 1.2. 개발 목표 및 주요 내용
본 프로젝트의 목표는 개인정보 유출 안내 문자, 이메일, 공지문을 받은 사용자가 복잡한 안내문을 직접 해석하지 않아도 자신의 상황을 빠르게 이해하고 필요한 조치를 선택할 수 있도록 돕는 AI 기반 웹서비스인 "LeakCare"를 개발하는 것이다. 많은 사용자는 유출 안내문을 받아도 법적 표현과 기술적 용어 때문에 내용을 정확히 이해하지 못하거나, 무엇을 해야 할지 몰라 안내문을 넘기는 경우가 많다. LeakCare는 사용자가 수신한 유출 통지문, 문자, 이메일을 AI로 분석하여 기업명을 추정하고, 관련 유출 사고 검색 및 유출 항목 확인을 거쳐 2차 피해 위험도를 분석한 뒤 최종적으로 대응 체크리스트를 생성하는 흐름으로 결과를 제공한다. 이를 통해 사용자는 자신이 받은 안내가 실제 유출 사고와 연관된 것인지 직관적으로 파악하는 동시에 비밀번호 변경, 2단계 인증 설정과 같은 기술적 조치부터 지인 사칭 예방을 위한 가족 공유까지 필수 행동 지침을 즉각적으로 이행할 수 있다. 

#### 1.3. 세부내용
LeakCare는 단순히 입력된 문장을 요약하는 서비스가 아니다. 사용자가 입력한 문자나 메일에서 AI가 기업명, 서비스명, 사고 키워드, 유출 항목을 추출하고, 이를 바탕으로 검색 API를 통해 관련 기사, 기업 공식 공지, 개인정보 유출 관련 자료를 조회한다. 이후 입력문과 외부 자료를 함께 비교하여 해당 안내가 실제 사고와 관련 있는지, 어떤 정보가 노출되었는지, 사용자가 주의해야 할 후속 위험은 무엇인지 판단한다.
최종적으로 서비스는 사용자가 확인해야 할 핵심 내용을 한 화면에 제공한다. 결과 화면에는 추정 기업명, 관련 사고 검색 결과, 유출 가능 항목, 위험 유형, 판단 근거, 우선순위별 행동 목록, 의심 문자 확인 결과, 가족 공유용 안내문, 신고·상담용 요약문이 포함된다. 이를 통해 사용자는 여러 사이트를 직접 검색하거나 전문 용어를 해석하지 않아도 현재 상황에서 어떤 행동을 먼저 해야 하는지 쉽게 파악할 수 있다.
서비스의 기본 분석 흐름은 다음과 같다. 먼저 사용자가 유출 안내 문자나 이메일을 입력하면 시스템은 전화번호, 이메일, 주소 등 민감정보를 자동으로 마스킹한다. 이후 AI가 기업명과 유출 사고 키워드를 추출하고, 검색 API가 관련 기사와 공식 공지를 조회한다. AI는 입력문과 검색 결과를 종합하여 사고 관련성과 유출 항목의 신뢰도를 판단하고, 스미싱, 피싱, 택배 사칭, 계정 탈취 등 사용자가 주의해야 할 위험을 분석한다. 마지막으로 사용자가 바로 실행할 수 있도록 비밀번호 변경, 2단계 인증 설정, 의심 링크 클릭 금지, 공식 고객센터 확인 등 필요한 조치를 우선순위에 따라 제시한다.

#### 1.4. 기존 서비스 대비 차별
| 구분 | 기존 서비스 | LeakCare |
| --- | --- | --- |
| 분석대상 | 유출 여부 조회 | 일반 신고 중심(제공받는 정보의 한계) | 사용자가 실제 받은 문자, 메일, 공지문 |
| AI 활용 | 제한적 또는 없음 | 기업명 추출, 사고 검색, 위험도 분석.대응 생성 |
| 외부 정보 활용 | 사용자가 직접 검색 필요 | 검색 API로 관련 기사, 공지 자동 조회 |
| 결과 제공 방식 | 일반 보안 수칙 안내 | 개인별 위험도와 대응 체크리스트 제공 |
| 후속 위험 안내 | 사용자가 직접 판단 및 조치 | 스미싱, 피싱, 택배 사칭 등 위험 유형, 자동 분석 |

| 확장 기능 | 신고, 조회 중심 | 가족 공유문, 신고용 요약문,의심 문자 분석 제공 |

LeakCare는 단순 조회·신고 중심의 기존 서비스와 달리 개인정보 유출 안내문의 해석부터 위험 분석, 맞춤형 대응 방안 제공까지 지원하는 AI 기반 통합 대응 서비스라는 점에서 차별성을 가진다.

#### 1.5. 사회적가치 도입 계획
본 프로젝트는 AI 기반 분석 기술을 활용하여 복잡한 유출 안내문을 이해하기 쉬운 형태로 제공하고, 유출된 개인정보 항목과 예상 위험을 분석하여 개인별 맞춤 대응 방안을 제시한다. 이를 통해 사용자가 신속하게 비밀번호 변경, 2단계 인증 설정, 의심 연락 차단 등 필요한 조치를 수행할 수 있도록 지원하여 개인정보 유출로 인한 2차 피해 발생 가능성을 낮추고자 한다.

또한 디지털 환경에 익숙하지 않은 고령층 및 정보 취약계층도 쉽게 활용할 수 있도록 직관적인 설명과 단계별 행동 가이드를 제공함으로써 디지털 정보 접근성 향상에 기여한다. 더불어 가족 공유 기능을 통해 개인정보 보호 정보를 주변인과 함께 확인하고 대응할 수 있는 환경을 조성하여 개인 차원의 대응을 사회적 예방 활동으로 확장하고자 한다.

궁극적으로 LeakCare는 개인정보 유출 사고 발생 이후의 대응 공백을 최소화하고, 사용자의 개인정보 보호 역량을 향상시키며, 안전하고 신뢰할 수 있는 디지털 환경 조성에 기여하는 것을 목표로 한다.

#### 1.6 법적 보안적 책임 범위
##### 1.6.1 기업에 대한 명예훼손 및 업무방해 책임 방지
AI가 특정 기업을 오인하여 개인정보 유출 기업으로 단정할 경우, 해당 기업으로부터 민·형사상 소송을 당할 리스크가 존재한다. 형법 제314조(업무방해)는 허위 사실 유포 등으로 타인의 업무를 방해한 자를 처벌하도록 규정하고 있으며, 정보통신망법 제70조(명예훼손) 역시 비방 목적으로 거짓 사실을 드러내어 명예를 훼손한 자를 가중 처벌하고 있다.
이러한 법적 성립 요건을 피하기 위해 AI 답변 생성 단계에서 특정 기업명을 단정적으로 확정 짓는 표현을 철저히 배제한다. 프롬프트를 제한함으로써 '위험도 분석 및 사칭 가능성 고지' 형태로 전환한다.
더불어 실제 뉴스나 한국인터넷진흥원(KISA)에 공식 보도된 유출 사건 데이터와 비교 분석을 수행한다. 공식 보도 사실이 확인된 경우에만 최근 유출 사건과의 유사성을 안내하고, 보도된 적이 없는 경우에는 기업 사칭 스미싱 확률이 높음을 분리하여 안내함으로써 기업의 부당한 피해와 법적 분쟁을 원천 차단한다.

##### 1.6.2 민사상 손해배상 책임 면제 
AI의 오탐이나 잘못된 대응 시나리오 제안으로 인해 사용자가 잘못된 조치를 취하거나 기업에 손해가 발생했다고 주장할 가능성이 있다. 이는 민법 제750조(불법행위의 내용)에 따른 고의·과실로 인한 손해배상 책임 문제로 이어질 수 있다.
이에 대응하여 민법 제105조에 규정된 당사자 간 합의 우선 원칙에 따라 유효한 약관 및 면책고지 시스템을 설계한다. 서비스 이용 전 팝업창과 안내문을 배치하여 "본 AI 분석 결과는 입력된 문자 메시지를 기반으로 한 추정치 및 참고용 정보이며, 실제 해당 기업의 공식 공지나 법적 확인 내용과 다를 수 있다"는 점을 명시한다. 이를 통해 민법 제750조에 따른 서비스 제공자의 고의·과실 가능성을 방어하고 최종 판단 책임이 사용자에게 있음을 분명히 한다.
또한 AI가 제시하는 비밀번호 변경, 계좌 지급정지 등의 대응 시나리오 역시 독립적인 법적 조언이 아닌 '권장 조치 사항'임을 인지시킨다. 이를 위해 KISA 보호나라, 금융감독원, 경찰청 등 정부 기관의 공식 행동 요령 및 신고 링크를 반드시 함께 제공하여 서비스가 독자적으로 내린 결론이 아님을 증명하고 책임의 범위를 제한한다.

##### 1.6.3 웹사이트 자체의 2차 개인정보 유출 및 데이터 보안 책임
사용자가 유출 문자를 복사하여 붙여넣는 과정에서 실명, 전화번호, 계좌번호 등 실제 개인정보가 웹사이트 서버로 유입된다. 이를 가림 처리 없이 외부 LLM API 업체 등으로 전송할 경우, 개인정보 보호법 제17조(개인정보의 제공) 위반에 해당하여 서비스 운영자가 직접적인 법적 책임을 지게 되는 심각한 보안 리스크가 발생한다.
이러한 위법 행위를 방지하고 데이터 보안적 책임을 다하기 위해, 사용자가 문자를 입력하는 즉시 외부 API로 전송되기 전 웹사이트 서버 내부에서 정규표현식 기반의 가림 처리(마스킹) 엔진을 거치도록 기술적 결합을 완료한다. 식별 가능한 개인정보를 서버 단에서 완벽하게 제거한 후 정제된 텍스트만 외부 LLM으로 송신하므로 '개인정보의 제3자 제공' 행위 자체가 성립하지 않도록 전제를 차단한다.
동시에 API 키 등 주요 인증 정보를 서버 환경변수(dotenv 등)로 분리하여 관리하는 방식을 적용한다. 이는 개인정보 보호법 제29조(안전조치의무) 및 동법 시행령에서 요구하는 '비밀번호 및 인증정보의 안전한 저장·관리' 규정을 충족하는 기술적 보호조치이며, 마스킹 엔진의 철저한 관리와 함께 본 서비스의 데이터 보안 성능 및 법적 정당성을 확보하는 핵심 기반이 된다.

### 3. 개발결과
#### 3.1. 전체시스템 흐름도

```text
[1] 사용자 입력
    → 개인정보 유출 문자, 이메일, 공지문 또는 의심 문자 입력
        ↓
[2] 프론트엔드 처리
    → 입력 내용을 백엔드 API로 전달
        ↓
[3] 개인정보 마스킹
    → 전화번호, 이메일, 주소 등 민감정보를 API 호출 전 자동 마스킹
        ↓
[4] AI 핵심 정보 추출
    → 기업명, 서비스명, 사고 키워드, 유출 항목 추출
        ↓
[5] 검색 API 연동
    → 기업명과 사고 키워드로 관련 기사, 공식 공지, 웹문서 검색
        ↓
[6] AI 종합 분석
    → 입력문과 외부 자료를 비교하여 사고 관련성, 위험도, 2차 피해 가능성 판단
        ↓
[7] 분석 결과 제공
    → 위험도, 유출 항목, 판단 근거, 대응 체크리스트, 가족 공유문, 신고용 요약문 제공
```

#### 3.4. 디렉토리 구조

LeakCare는 Next.js 기반 웹서비스 구조로 설계하며, 화면 페이지, 백엔드 API Route, AI·검색 API 연동 로직, 개인정보 마스킹 유틸리티를 역할별로 분리한다.  
API 키와 민감정보는 GitHub에 업로드하지 않고 `.env.local` 및 배포 환경변수로 관리한다.

LeakCare/
├── public/                     # 로고, 예시 이미지 등 정적 파일
├── src/
│   ├── app/                    # 메인 페이지 및 전역 스타일
│   ├── components/             # 입력창, 위험도 카드, 체크리스트 등 UI 컴포넌트
│   ├── data/                   # 시연용 Mock 분석 결과 데이터
│   ├── utils/                  # Mock 분석 함수 및 보조 함수
│   └── types/                  # 분석 결과 타입 정의
├── docs/                       # 시스템 구성도, 발표자료, 시연 자료
├── README.md
├── package.json
├── .env.example
└── .gitignore

> 실제 API 키가 포함된 `.env.local`은 GitHub에 업로드하지 않으며, 저장소에는 `.env.example`만 포함한다.
